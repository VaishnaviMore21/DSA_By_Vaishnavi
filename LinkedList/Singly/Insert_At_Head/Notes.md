# Insert_At_Head

## Problem statement
Implement Insert_At_Head using LinkedList patterns.

## Intuition
Use node pointer operations and traversal carefully to preserve list structure.

## Brute force
Create new node and make it head.

## Better
Same as brute; direct pointer update.

## Optimal
Head insertion is already optimal O(1).

## Dry run
Take sample list and trace pointer updates step by step at each node change.

## Edge cases
Empty list, single node list, and minimal valid length inputs.

## Time complexity
O(1)

## Space complexity
O(1) extra for iterative pointer-based methods unless hashing/recursion used.

## Interview notes
Remember to return new head.
