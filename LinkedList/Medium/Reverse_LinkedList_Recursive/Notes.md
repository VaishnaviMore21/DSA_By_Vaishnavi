# Reverse_LinkedList_Recursive

## Problem statement
Implement Reverse_LinkedList_Recursive using LinkedList patterns.

## Intuition
Use node pointer operations and traversal carefully to preserve list structure.

## Brute force
Collect values then write reverse.

## Better
Recursive backtracking reversal.

## Optimal
Recursive is elegant; iterative has less stack.

## Dry run
Take sample list and trace pointer updates step by step at each node change.

## Edge cases
Empty list, single node list, and minimal valid length inputs.

## Time complexity
O(n)

## Space complexity
O(n) recursion stack

## Interview notes
Recursion adds call stack space.
