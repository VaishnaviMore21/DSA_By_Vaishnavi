# Linear Search

## Problem statement
Find index of target element, else return -1.

## Intuition
Check elements sequentially.

## Brute force explanation
Single linear scan.

## Better approach explanation
No better approach in unsorted array without preprocessing.

## Optimal approach explanation
Brute force and optimal are same here.

## Dry run
[4,2,7], target=7 -> index 2.

## Edge cases
Empty array, target absent.

## Interview tips
Mention binary search requires sorted data.
